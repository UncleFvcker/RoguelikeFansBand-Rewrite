# 简体中文本地化风格指南 v1

## 语气与人称

- 玩家行动默认使用“你”，不使用“玩家”“角色”代替第二人称。
- 系统提示保持简洁、客观；战斗消息允许轻微叙事感，但不使用网络流行语。
- 内容名称不自动添加“一个”“某个”等数量词，数量和语法由完整消息模板负责。

## 标点与空格

- 中文句子使用全角逗号、句号、冒号、分号和括号。
- 中文与阿拉伯数字之间通常保留一个半角空格，例如“造成 3 点伤害”。
- 按键名、文件扩展名、稳定 ID 和产品名保持半角形式，例如 `G`、`.rfbsave`、`PixiJS`。
- 乘号统一使用 `×`，不使用字母 `x` 代替。

## 参数与语序

- 每条 Fluent 消息必须是完整句子，禁止从 TypeScript 或 Rust 传入“的”“了”“被”等语法碎片。
- 具名参数表达语义，例如 `$target`、`$damage`、`$quantity`；不得使用 `$value1`、`$text2`。
- 中文可自由调整参数顺序，不追随英文模板。示例：英文先物品后数量，中文可以先数量后物品。
- 复数和数量选择由 Fluent 表达式处理，业务代码不拼接“个”“堆”“次”。

## 内容名称

- 原创怪物、物品和地形名称必须同时提供 `en-US` 与 `zh-CN` key。
- 逻辑判断只使用稳定内容 ID；显示名称不能反向参与规则判断。
- 未知对象使用统一回退名称，不显示内部 ID，除非处于开发诊断模式。

## 按键与界面

- 按键名使用大写拉丁字母，如 `G`、`WASD`、`HJKL`。
- 面板标题优先使用清楚的功能名称；技术验证场可以保留 `HTML`、`PixiJS` 等架构标记。
- 语言切换不得改变回合、命令序号、RNG、存档或 state hash。
