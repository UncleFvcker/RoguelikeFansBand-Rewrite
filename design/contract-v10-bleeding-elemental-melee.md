# Contract v10：流血与内容驱动元素近战

状态：active contract baseline

## 1. 目标

contract-v10 在 contract-v9 的状态/抗性底座上增加两个实际规则入口：流血周期伤害，以及由角色内容定义驱动的近战伤害类型。这样元素抗性不再只存在于纯函数测试，而能真实影响怪物攻击。

## 2. 流血

- 稳定状态 ID：`rfb.status.bleeding`；
- 每个 `worldTick` 按状态强度造成物理伤害；
- 多个周期状态按状态 kind ID 排序，因此同一 tick 中流血先于毒素；
- 伤害结算后统一衰减持续时间，再生成过期事件；
- 状态致死继续立即中止对应行动队列。

旧 RFB 的割伤使用连续严重度、体质恢复与按能量缩放伤害。当前原创垂直切片先固定“强度即每 tick 基础伤害”，保留稳定顺序和未来严重度映射接口，不复制旧阈值表或 C 插值实现。

## 3. 内容驱动元素近战

`ActorDefinition` 新增 `damageType`：

- `physical`、`acid`、`electricity`、`fire`、`cold`、`poison`；
- 旧内容缺失时默认 `physical`；
- 物理怪物近战先经过现有 AC 减伤，再经过抗性 profile；
- 元素怪物近战跳过物理 AC 减伤，直接按对应抗性等级结算；
- `DamageDiceDto` 同时输出骰子和伤害类型。

原创余烬微光改为火焰近战。玩家火抗 50% 按确定性整数规则减伤，火免疫把伤害降为 0。
