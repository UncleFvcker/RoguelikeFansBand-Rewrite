# Contract v25：怪物携带物与统一死亡掉落事务

状态：协议 1.25 / contract-v25 active baseline

## 已完成边界

- 物品运行位置新增 `CarriedBy { actorId }`。怪物定义可通过独立 `carriedLootTableId` 声明出生携带物，同时保留 `lootTableId` 表示死亡时生成的普通掉落，两个字段语义不混用。
- 创建世界时按稳定怪物实例 ID 顺序生成携带物；每个 roll 继续固定按物品、品质、词条消耗三次权威 RNG，并立即分配稳定 `generated.item.N`。
- 怪物携带物不进入地面、玩家背包或装备 DTO，不计入玩家负重，也不在怪物生前泄露种类、质量或词条。
- save v1 新增可选 `carriedItems` 列表，保存物品真值与持有者 actor ID。载入会拒绝不存在或非怪物的持有者、重复实例 ID、非法数量、质量和词条组合。
- 玩家近战、射击、投掷和持续状态四条死亡路径统一进入同一个死亡结算入口。结算先按实例 ID 放下真实携带物，再按 contract-v24 规则生成普通死亡掉落；携带物保留原实例 ID、质量、词条和知识状态，死亡本身不会重新抽取它。
- 余烬微光出生时固定携带一枚优良、带“谐振锋芒”词条的回声护符，死亡时仍会执行原有加权普通掉落。

## 与原版 RFB v1.3.0.7 对比

原版 `object_type.held_m_idx` 保存物品持有怪物索引，怪物通过 `hold_o_idx` 维护携带物链；`src/xtra2.c:1154` 附近的 `monster_death()` 会先放下真正携带的物品，随后在 `src/xtra2.c:2365` 附近生成普通死亡掉落。

相同点：

- 区分“已经属于怪物的真实物品”和“死亡时生成的普通掉落”；
- 怪物死亡后，真实携带物保留自身属性并落在死亡格；
- 携带物层可以继续承载偷窃、缴械和怪物拾物。

主动差异：

- 重构使用稳定 actor/item 字符串 ID 和显式 `ItemLocation`，不保存数组索引或对象链表；
- 携带物和普通掉落均有固定 RNG 顺序、稳定事件顺序、存档不变量和回放检查点；
- 当前尚未实现偷窃、缴械、怪物主动拾物、掉落次数 flag、主题掉落和特殊怪物硬编码。

本切片需要兼容测试，但目标是规则关系与原版一致，不追求原版数值或随机序列一致。
